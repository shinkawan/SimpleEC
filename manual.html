1. プラグイン概要

このプラグインはWordPress上で汎用的な商品を販売するための機能をすべて提供します。

✅ 主な機能

機能 説明
商品販売 ダウンロード版・配送品の2形式に対応
決済 クレジットカード・PayPay・銀行振込・代金引換
クーポン 割引コード（定額・定率）の発行と管理
注文管理 受注一覧から全注文を一括管理
メール通知 注文確認・キャンセル・発送通知メールを自動送信
ダウンロード管理 デジタル販売の有効期限・回数制限
CSV出力 売上データをCSVエクスポート（確定申告対応）
注文照会 購入者が注文番号とトークンで詳細を確認可能
かご落ちリカバリー 未完了のカートを自動検知し、リマインドメールを送信
送料計算 住所入力後に自動計算（通常配送商品のみ）
在庫アラート 設定した在庫数を下回った際に管理者にメール通知
会員割引 ログイン中の会員に対して自動的に一律割引を適用
再購入 過去の注文履歴からワンクリックでカートに追加（リピート促進）

2. 初期設定（各種設定）

WordPress管理画面 → 各種商品 → 各種設定 にある 5 つのタブで設定を行います。

📌 設定項目（各種設定）

タブ 項目 説明
一般設定 受注通知メール送信先 新しい注文が入った際に通知を受け取るメールアドレスを入力します。
表示用店舗名 / メール 注文メールの送信元（名前・アドレス）として使用されます。
ギャラリーのカラム数 [ec_gallery] で表示する商品の横並び数を設定します。
在庫アラートのしきい値 在庫がこの数値を下回った際に管理者にアラートメールを送信します。
会員割引率 (%) 会員ログイン時に適用する割引率を設定します（例: 10）。0で無効。
マイページ設定 [ec_member_dashboard] を設置した固定ページを選択します。[v3.12.2]
決済設定 利用を許可する決済 カード(Stripe)、PayPay(Stripe)、銀行振込、代金引換のON/OFF。
Stripe API / Webhook Stripe決済に必要なキーを設定します。Webhookシークレットは安全な正規の決済完了通知（特にサブスク解約同期）のために必須です。
デジタル販売設定 ダウンロード商品の有効期限（日数）と上限回数を設定します。

配送業者名 / 日数目安 日本郵便、ヤマト運輸など（追跡番号の案内時や注文照会時に活用されます）。

各決済の手数料説明 「振込手数料はお客様負担です」等の補足テキストを編集します。

SNS連携 Google / LINE 連携 SNSログインボタンの表示切替と、各サービスのAPIキーを設定します。
会員規約 規約本文の入力 会員登録や購入時に適用される規約を編集します。[ec_membership_terms]で表示。

✅ インボイス制度（適格請求書）への対応

「特定商取引法」タブで 登録番号（T番号） を設定すると、以下の場所に「登録番号：T123...」および「10%対象 / 8%対象」の税率別内訳が自動的に表示されます。

  • お客様への自動返信メール（注文確認・完了・発送）: 会員割引額を別途明記し、明細合計と一致します。
  • 管理者用「受注編集」画面 および 管理者向通知メール
  • 帳票出力（納品書・領収書）: 「会員特別割引」行が追加され、請求額と完全一致します。
  • 購入者向けの「注文照会ページ / マイページ」

🚛 都道府県別送料の詳細設定（一括操作）

「送料設定」タブの下部にある都道府県一覧から、地域ごとに送料をカスタマイズできます。

  • チェックを入れて適用: 左側のチェックボックスで選択し、上部の「一括操作」バーで金額を入力して適用できます。
  • 地域一括ボタン: 「関東」「九州」などの地方見出しの右側にある入力欄を使うと、その地方の全県にまとめて同じ送料を適用できます。
  • 一律送料への併用: 空欄の県は「全国一律送料」が適用されます。

🔗 Stripe Webhook（署名シークレット）の設定手順

決済の完了やサブスクリプション（定期購入）の自動更新・解約をサイトに正確に同期させるために必須の設定です。

  1. エンドポイントの作成: StripeダッシュボードのWebhook設定を開き、「エンドポイントを追加」をクリックします。
  2. URLの入力: エンドポイントURLに 【あなたのサイトURL】/?photo_purchase_action=stripe_webhook を入力します。
  3. イベントの選択: 以下の3つを選択して保存します。
    • checkout.session.completed （決済完了時）
    • invoice.paid （サブスクリプション自動更新時）
    • customer.subscription.deleted （サブスクリプション解約時）
  4. シークレットの取得: 追加されたエンドポイントの設定画面で「署名シークレット」を表示し、whsec_ から始まる文字列をコピーします。
  5. プラグインへの登録: WordPress管理画面「各種設定」＞「決済設定」タブ内の Webhook署名シークレット 欄にペーストして保存します。

3. 商品の登録・管理

各種商品 → 商品を新規追加
タイトル、商品ファイル、サムネイル画像、販売形式と価格を設定します。
在庫管理の設定
「在庫管理を行う」にチェックを入れると、在庫数に基づいて販売を制限。0になると自動で「売り切れ」表示になります。

3.1 SKUバリエーション管理 (v4.1.0～)

「サイズ」や「カラー」ごとに異なる価格や在庫を設定する場合に使用します。本格的な物販やアパレル販売に最適です。

⚙️ 設定手順

1
バリエーションの有効化
商品編集画面の「在庫・価格設定」パネルにある「バリエーションを使用する」にチェックを入れます。
2
属性の定義
「属性の定義」エリアに、属性名（例: カラー）と値（例: 赤, 青, 黄）を入力します。複数の属性（例: サイズとカラー）を組み合わせることも可能です。
3
組み合わせの自動生成
「バリエーションを自動生成」ボタンを押すと、すべての組み合わせがリスト化されます。
4
一括設定ツールの活用（時短！）
生成された大量の項目を一つずつ入力するのは大変です。リスト上部の「一括設定 (Bulk Actions)」エリアを使いましょう。
  • 一括価格設定: すべてのバリエーションに一律の加算額を設定。
  • SKU一括生成: 「ITEM-RED-」等のプレフィックスを入力すると、連番のSKU（ITEM-RED-001など）が自動付与されます。
5
個別の微調整・保存
特定のサイズだけ在庫を減らしたり、価格を変えたりした後、商品の「更新」ボタンを押して保存します。
💡 フロントエンドの表示について
設定を保存すると、商品詳細ページでは「カラー」や「サイズ」が個別のセレクトボックスとして表示されます。在庫切れの組み合わせを選択すると、カートボタンが自動的に「売り切れ」に切り替わります。

4. メディアライブラリからの一括登録

数十点、数百点の商品を効率よく登録するための機能です。

📝 手順

  1. WordPressメニュー：各種商品 → 一括登録 を開く
  2. 「メディアライブラリから選択」をクリックし、画像を選択
  3. カテゴリーと各形式の初期価格を設定して「商品を一括生成する」をクリック

5. 注文管理（受注一覧）

WordPress管理画面 → 各種商品 → 受注一覧 で全注文を管理します。

📋 注文のフィルタリング

上部のタブで表示する注文を切り替えられます。

  • 通常注文: 未決済の電子決済（Abandoned）を除いた、有効な注文のみを表示します。
  • 放置注文: 決済が完了していないStripe/PayPay注文、または7日以上経過した入金待ち注文を抽出します。
  • すべて表示: システム上のすべての注文を表示します。

⚙️ ステータスの一括更新

複数の注文を選択して、一括でステータスを変更できます。

  1. 受注一覧で対象の注文にチェックを入れる
  2. 上部または下部の「一括操作」から以下を選択：
    • ステータス: 準備中に変更
    • ステータス: 完了（発送済み）に変更
  3. 「適用」ボタンをクリック

※ステータス変更時には、購入者へ自動的に「ステータス更新通知メール」が送信されます。

🗑️ 選択して一括削除

「一括操作」から「削除」を選んで「適用」をクリックすることで、複数の注文をまとめて削除できます。この操作は元に戻せませんのでご注意ください。

6. 商品ラベル（リボン）の設定 [v3.11.0]

特定の商品を際立たせるためのカスタムラベル（リボン）を、画像の上に表示できます。

📝 設定方法

  1. 商品編集画面の「商品の詳細設定」を開きます。
  2. 「商品ラベル」欄に表示したい文言（例：「NEW」「SALE」「おすすめ」等）を入力します。
  3. カラーピッカーやクリック選択ボタンを使用して、ラベルの背景色を自由に変更できます。
  4. 「更新」ボタンを押すと、ショップ画面の商品画像左上に指定した色でリボンが表示されます。
  5. 空欄にするとラベルは非表示になります。

※新着商品や期間限定アイテムの訴求に非常に効果的です。

7. 再入荷通知機能 [v3.11.3]

売り切れ商品の在庫が復活した際に、希望するお客様へ自動でメールを送信できます。

📝 仕組みと活用方法

  • お客様側の操作: 売り切れ商品に表示される「再入荷通知を受け取る」ボタンからメールアドレスを登録します。
  • 管理側の確認: 商品編集画面の「売り切れ設定」箇所に、現在何名が通知を待っているか表示されます。
  • 自動送信トリガー: 商品の在庫数を増やすか、「売り切れにする」のチェックを外して保存した瞬間に、登録者全員に通知メールが即座に一括送信されます。
  • [v3.11.3] 高精度なリンク: 通知メール内の「商品を確認する」ボタンは、お客様が登録を行った際のギャラリーURL（例: サブページ /ndp/ 等）を記憶して生成されるため、迷わず購入ページへ誘導できます。

※通知送信後は待機リストが自動的にリセットされます。機会損失を防ぎ、売上アップに直結する機能です。

8. 配送状況の追跡連携

送り状番号と配送業者を紐付けることで、購入者がワンクリックで追跡ページを確認できるようになります。

🔗 連携の仕組み

  • 発送通知メール: メール内の送り状番号が自動的に追跡サイトへのリンクになります。
  • 注文照会ページ: 照会結果画面に「配送状況を確認（外部サイト）」ボタンが表示されます。
✅ 主要4社（ヤマト・佐川・郵便・西濃）の追跡URLに標準対応しています。

9. 会員機能とマイページ [v3.10.0]

購入履歴の確認、配送先情報の管理、サブスクリプションの管理ができる統合マイページ機能です。

🛡️ パスワード不要の安全なログイン (OTP)
セキュリティ向上のため、パスワードの設定・管理を完全に廃止しました。登録メールアドレスに届く 「6桁の認証コード（ワンタイムパスワード）」 を入力するだけで、安全かつ簡単にログインできます。

👤 会員情報の自動保存と活用

ログイン状態で商品を購入すると、入力した配送先情報（住所・電話番号など）が自動的に会員情報として保存されます。

  • 次回の入力省略: 次回購入時、保存された住所が自動的に入力欄に反映され、手間が大幅に削減されます。
  • 情報の変更: マイページ内の「会員情報・配送先設定」から、いつでも住所や電話番号を更新できます。

🚀 マイページでできること

  • 購入履歴の一覧: 過去の全注文内容、合計金額、支払い方法を確認。
  • 配送先設定の管理: 登録されている配送先情報の確認と編集。
  • 配送状況の確認: 発送済みの注文について、送り状番号の確認と追跡サイトへの移動。
  • デジタル商品の再ダウンロード: 期限内であれば、ここから直接ファイルをダウンロード可能。
  • サブスクリプション管理: Stripeと連携し、クレジットカードの更新やプランの解約をセルフサービスで行えます。

💎 会員限定割引 [v3.12.0]

特定の会員（ログインユーザー）に対して、ショップ全体の商品を自動的に割引価格で提供する機能です。

✨ プレミアムな表示と自動計算

  • 一律割引: 設定した割引率（%）が、サブスクを除くすべての商品に自動適用されます。
  • プレミアムバッジ: ログインユーザーには、**専用のグラデーションバッジ**（インディゴ〜パープル）が表示され、特別感を演出します。
  • 打ち消し線表示: 元の価格に打ち消し線を引き、その隣に目立つ会員価格を自動で並記します。
  • リアルタイム反映: 商品詳細だけでなく、カートドロワーやチェックアウト画面でも常に割引後の最終金額が表示されます。
  • システム全域への統合 [v3.12.5]: 購入者へのメール、納品書、領収書、管理用CSV、およびマイページ履歴のすべてにおいて「会員割引額」が明記され、計算不整合が解消されました。
💡 運営のヒント: マイページには「会員割引適用中」の専用バナーが表示され、会員であるメリットを常に意識させることができます。

12. 再購入（リピート購入）機能 [v3.12.0]

過去に購入した商品を、マイページからワンクリックで現在のカートに追加できるリピート促進機能です。

📦 数量の自動統合プロセスの追加

  • 「再購入」ボタン: 会員マイページの各注文履歴カードに表示されます。
  • 重複商品の統合（マージ）: カートにすでに同じ商品が入っている場合、自動的に**既存の数量に追加（マージ）**されるため、カート内が重複した商品で溢れるのを防ぎます。
  • スマートなフィードバック: ボタンを押すと即座に「✅ X個追加しました」とボタン上に表示され、自動的にカートドロワーが展開します。
✅ 時短体験: 消耗品やお気に入りのアイテムを「以前と同じ条件（オプション等）」で即座にカートへ入れることができ、チェックアウトまでの工数を最小限に抑えます。

9. SNSログイン連携

GoogleおよびLINEアカウントを使用して、ワンクリックでログイン・会員登録が可能です。

🛠️ 管理者側の設定手順

  1. 各種設定 → SNS連携 タブを開きます。
  2. SNSログインを有効にする チェックボックスで、機能全体のON/OFFを切り替えられます。
  3. Google Cloud Console または LINE Developers で作成した クライアントID と クライアントシークレット を入力します。
  4. 設定画面に表示されている「コールバックURL」を、各SNS側の設定（承認済みのリダイレクトURI）に登録してください。
💡 シンプルな会員登録: 初めてSNSでログインした際、自動的にWordPressアカウントが作成されます（メールアドレスが連携されます）。

10. システムログの確認

決済エラーやWebhookの失敗など、システムの稼働状況をリアルタイムで確認・管理できます。

📋 ログの閲覧方法

WordPressメニュー：各種商品 → システムログ を開きます。

  • 最新100件を表示: 発生日時、エラーレベル（ERROR/INFOなど）、メッセージ、および詳細なデータが一覧で表示されます。
  • エラーの特定: StripeのWebhookが認証できなかった場合や、メール送信に失敗した際などの原因調査に役立ちます。
  • ログの消去: 定期的に「ログをすべて消去」ボタンでデータをリセットできます（最新500件を超えると自動的に古いものから削除されるため、肥大化の心配はありません）。
💡 「ログはありません」と表示される場合: システムが正常に稼働し、エラーが一度も発生していない状態です。エラーが発生した際のみ記録されます。

11. 会員規約（利用規約）の設定

会員登録や商品の購入に際して、お客様に同意いただく規約を管理・表示できます。

📋 設定と表示の手順

  1. 各種設定 → 会員規約 タブを開きます。
  2. 規約の本文を入力し「設定を保存」をクリックしてください。
    • ショップ名の自動反映: 本文中に [ショップ名] または [ec_shop_name] と記述すると、WordPressの「サービス名（サイト名）」が自動的に挿入されます。
    • 日本語のショートコードが不安な場合は、英数表記の [ec_shop_name] をご利用ください。どちらも同じ動作をします。
    • HTMLタグ（p, br, strong等）が使用可能です。
  3. 固定ページの自動生成: プラグインのインストール・有効化時に、[ec_membership_terms] が配置された「会員規約」固定ページが自動的に作成されています。
  4. もしページを削除してしまった場合は、再度固定ページを作成し、以下のショートコードを貼り付けてください。
    [ec_membership_terms]
💡 同意の仕組み: Simple ECでは、ユーザーの利便性を考慮し「ログインまたは注文確定を行う＝規約に同意した」とみなす法的構成を採用しています。ログインフォームや注文ボタン付近に、規約へのリンクを含む文言が自動的に挿入されます。

13. 注文ステータスフロー

ステータス 意味 次のアクション
入金待ち 銀行振込の入金確認待ち 入金確認後 → 「決済完了」へ変更
決済完了 / 発送待ち 決済が完了し、配送準備中 発送完了後 → 「発送済み」へ変更
🟣 サービス中 / 継続中 サブスクリプション（定期購入）が有効な状態 次回の自動課金を待ちます。解約されるまでこの状態が続きます。
発送済み / 完了 商品を発送済み、または単発サービスの提供完了 —（完了）
キャンセル 注文をキャンセルした 購入者に自動でキャンセルメールが送られます
💡 クレジットカードまたはPayPay（Stripe）決済の場合、決済が完了した時点で自動的に「決済完了 / 発送待ち（サービス・デジタルの場合は『サービス中 / 継続中』）」になります。

14. 各種アクション操作

編集 — 注文を編集

購入者情報・ステータス・送り状番号を変更できます。送り状番号を入力して保存すると、購入者への発送通知メールが自動送信されます。

📧 再送 — 確認メール再送

注文確認メールを購入者に再送します。メールが届いていないと連絡を受けた際に使用します。

キャンセル — 注文キャンセル

注文をキャンセルします。実行すると：

  • ステータスが「キャンセル」に更新される
  • 購入者にキャンセル通知メールが自動送信される
  • クレジットカード・PayPay（Stripe）決済の場合、自動的に全額返金が実行されます。
💡 自動返金の結果は画面上のメッセージで確認できます。銀行振込や代金引換の場合は、手動での返金対応が必要です。
💡 在庫とクーポンの復元：キャンセルを実行すると、その注文に含まれていた商品の在庫および使用されたクーポンの利用回数が自動的に復元（差し戻し）されます。
⚠️ キャンセル後に「発送済み」などに戻すには「編集」からステータスを変更してください。

📄 納品書 / 🧾 領収書 — 各種帳票出力

注文詳細に基づいた納品書や領収書を「印刷用A4レイアウト」で表示します。ブラウザの印刷機能を使用してPDF保存や紙への印刷が可能です。

  • 納品書：商品明細や配送先を記載します。
  • 領収書：合計金額を強調した領収書形式で出力します。

削除 — 注文削除

注文を完全に削除します。この操作は元に戻せません。通常はキャンセルを使用し、削除は不要な注文のみに限定することを推奨します。

💡 在庫・クーポンの復元：未キャンセルの注文を削除した場合、削除前に在庫およびクーポン使用回数が自動的に復元されます。

15. クーポン管理

WordPress管理画面 → 各種商品 → クーポン管理 でクーポンを作成・管理できます。

🎫 クーポンの作成

「新規追加」から以下の項目を設定してクーポンを発行します。

項目 説明
クーポンコード お客さまが入力する半角英数字のコード（例: WELCOME2024）
割引種別 百分率(%) または 固定額(円) を選択
割引額 10%なら「10」、500円なら「500」と入力
有効期限 クーポンの利用期限を設定（未設定で無期限）

🛒 お客さま側の利用方法

💡 利用回数の自動カウント：注文が確定するたびに、利用されたクーポンの「現在の利用回数」が自動で加算されます。上限に達したクーポンは自動的に無効となります（※注文キャンセル時には自動的に回数が差し戻されます）。

16. サブスクリプション機能 [v3.0.0]

継続的なサービス（月額・年額など）を販売するための機能です。

📦 商品をサブスクとして設定する

「商品を新規追加」または「編集」画面で、「この商品をサブスクリプションとして販売する」にチェックを入れます。

💡 サブスクを有効にした商品は、Stripe Checkout で自動的に「継続課金」として処理されます。
🚚 配送が必要な商品 (定期購入)にチェックを入れると、チェックアウト時に住所入力が必須になります（サービス提供型の場合はチェックを外してください）。
⚠️ サブスクリプション商品が含まれる場合、クレジットカード決済のみとなります（PayPay・銀行振込・代金引換はご利用いただけません）。

💳 購入者の管理ポータル

購入者は以下の方法で自身のサブスクリプションを管理（解約、カード変更など）できます。

  • 注文照会ページ: 照会結果の画面に表示される「サブスクリプション管理」ボタンから移動。
  • 統合マイページ: [ec_member_dashboard] を設置したページから（メール認証コードによる安全なログインが可能です）。
⚠️ Webhookの設定：サブスクの「解約」をシステムに反映させるには、Stripe管理画面でWebhookの設定が必要です。詳しくはStripe Webhook（署名シークレット）の設定手順をご参照ください。

20. 在庫アラート機能 [v3.11.2]

在庫切れによる販売機会の損失を防ぐため、在庫が少なくなった際に自動で管理者に通知する機能です。

🚨 通知の仕組み

  • しきい値の設定: 「設定 > 一般」タブの「在庫アラートのしきい値」で、何個以下になったら通知するかを設定できます（デフォルト: 5）。
  • リアルタイム判定: 購入によって在庫が減った際や、CSV・手動で在庫を書き換えた際に自動チェックが行われます。
  • メール通知: 在庫がしきい値以下になると、管理者の受信メールアドレスに「【在庫不足アラート】商品名」という件名のメールが届きます。
  • 重複防止: 一度数値が下がって通知された後は、在庫が補充（しきい値より大きく設定）されるまで、同じ商品（またはSKU）のアラートは二重送信されません。
  • SKU対応 [v4.2.4]: 通常在庫だけでなく、バリエーション（SKU）ごとの個別在庫も正確に監視され、不足したバリエーション名がメールに明記されます。
💡 運用のコツ
人気商品はしきい値を多め（例: 20個）に設定し、一点ものや入荷が早いものは少なめ（例: 2個）に設定したしきい値を「一般設定」で調整してください。

21. ディープリンク機能（特定商品への直リンク） [v3.11.3]

SNSやメルマガから、特定の商品の「詳細画面」を開いた状態で直接案内できる機能です。

🔗 リンクの形式

ギャラリーが設置されているページのURLの末尾に ?photo_id=商品ID を付与します。

https://example.com/gallery/?photo_id=123

✨ 動作の仕組み

  • 自動オープン: 指定されたページを開いた直後に、該当商品の詳細モーダル（クイックビュー）が自動的に開きます。
  • 自動スクロール: 商品がギャラリーの下の方にあっても、開いた瞬間にその商品の位置までスムーズにスクロールします。
  • 効率的な集客: SNSやメルマガからリンクされた際、ユーザーを探させる手間を省き、コンバージョン率を向上させます。
  • SEOとリダイレクトへの配慮: WordPress標準の「正規URLリダイレクト」に干渉せず、サブページでも正しく動作するよう最適化されています。

22. かご落ちリカバリー機能（自動リマインド） [v4.0.0]

カートに商品を入れたまま購入を完了せずに離脱したユーザーに対して、自動的にリマインドメールを送信し、購買率（コンバージョン率）を高める機能です。v4.0.0 では成果を可視化するダッシュボードと配信停止機能が追加されました。

🛠 設定方法

  1. WordPress管理画面から「すべての商品」＞「各種設定」を開きます。
  2. 「かご落ち対策」タブをクリックします。
  3. 「かご落ちリカバリーメールを自動送信する」にチェックを入れます。
  4. 送信タイミングを選択します（3時間後、24時間後、48時間後から選択可能）。

📊 成果の可視化 (統計ダッシュボード)

設定画面の下部、およびWordPressのメインダッシュボードにて、以下の指標をリアルタイムで確認できます。

  • リマインド送信数: 実際にユーザーに送信されたメールの総数。
  • クリック数 (率): メール内の復元リンクがクリックされた回数。
  • リカバリー成功 (率): メール経由でカートが復元され、最終的に購入が完了した数と金額。

🛡 プライバシーと配信停止

  • 配信停止リンク: リマインドメールの末尾には配信停止リンクが含まれており、ユーザーはワンクリックで次回以降の通知を止めることができます。
  • 遵守事項: ユーザーのプライバシーに配慮し、不要なリマインドを繰り返さないための仕組みが備わっています。

💡 仕組みとメリット

  • データの捕捉: ユーザーがメールアドレスを入力した（またはログインしている）状態でカートを操作すると、自動的にデータの同期が始まります。
  • 条件判定: 最後にカートを更新してから指定した時間が経過し、かつその間に注文が完了していない場合にのみメールが送信されます。
  • ワンクリック復元: 送信されるメールには「専用の復元リンク」が含まれています。ユーザーがリンクをクリックすると、以前追加した商品がカートに入った状態でサイトが開きます。
※注意: メールの送信には WordPress の WP-Cron 機能を使用しています。サイトへのアクセスが全くない時間は、送信時間が多少前後する場合があります。

23. 国際配送の設定と管理 [v5.0.0]

日本国内だけでなく、世界各国への配送に対応するための設定と運用方法について説明します。

⚙️ 設定方法

  1. 各種設定 → 送料設定 タブを開きます。
  2. 「海外への発送を有効にする」にチェックを入れます。
  3. 国別送料設定テーブルで、発送先の国名と送料を入力し「行を追加」して設定を保存します。
  4. 送料無料の除外設定: 「海外発送を送料無料の対象外とする」にチェックを入れると、国内向けの設定（例: 1万円以上で送料無料）に関わらず、海外発送には必ず設定した送料が適用されます。

🛒 購入者側の体験（チェックアウト）

  • 国選択: 支払い画面に「国名」セレクトボックスが表示されます。
  • 動的な入力欄: 「日本」以外を選択すると、都道府県の選択肢が消え、代わりに「州 / 地域 (State/Region)」の自由入力欄が表示されます。バリデーション（必須チェック）も自動で切り替わります。
  • 支払い方法の制限: 海外発送が選択された場合、代金引換（代引き）は自動的に選択肢から除外されます。万が一、不正な方法で選択された場合も、サーバー側で決済をブロックする安全設計となっています。
  • 通貨の明記: すべての決済は日本円 (JPY) で行われる旨の注釈が表示されます。

📦 管理者側の運用とリファイン

  • 受注一覧のバッジ: 海外からの注文には [国名コード] （例: [US]）の目立つバッジが表示され、国内注文と瞬時に判別できます。
  • 住所フォーマットの自動切替: メール通知や受注詳細、納品書などの住所欄が、その国に最適な並び順（海外なら「住所 → 州 → 郵便番号 → 国名」）に自動で組み替えられます。
  • 注文編集: 管理画面からの注文編集でも、国名に応じた入力欄の動的切り替えが適用されます。
  • 運用アドバイス (銀行振込): 海外からの送金手数料は国内と異なるため、各種設定の「各種決済の手数料説明」などに「海外からの送金手数料はお客様負担」である旨や、必要に応じてSWIFTコードなどの案内を追記しておくことを推奨します。
⚠️ Stripe決済の安全性: フロントエンド（ブラウザ）で算出された送料は参考用とし、実際の決済金額はサーバー側で再計算されます。これにより、悪意のあるユーザーによる送料の改ざんを完全に防止しています。

25. 会員規約の表示・自動生成

オンラインショップ運営に欠かせない会員規約を、簡単に管理・表示できる機能です。

📄 自動生成と表示

  • 固定ページの自動生成: プラグインを有効化した際、自動的に「会員規約」という固定ページが作成され、ショートコード [ec_membership_terms] が挿入されます。
  • デフォルト規約のセット: 初回インストール時に、標準的な第1条〜第12条からなる規約文がデータベースに保存されます。
  • ショートコード [ショップ名]: 規約本文内で [ショップ名] または [ec_shop_name] を使用すると、WordPressの設定にある「サイト名」が自動的に表示されます。

⚙️ 規約の編集方法

WordPress管理画面 → 各種商品 → 各種設定 → 会員規約 タブから、規約のタイトルと本文を自由に編集できます。HTMLタグも使用可能です。

💡 みなし同意について
ショッピングカートのチェックアウト画面やログイン画面には、「認証を行うことで、会員規約に同意したものとみなされます」という一文が自動的に表示されるようになっています。

27. セキュリティと信頼性

本プラグインは商用利用を前提として、以下のセキュリティ対策を講じています。

  • 価格操作の防止: チェックアウト時にブラウザ側で商品価格やオプション価格が書き換えられた場合でも、サーバー側でデータベースの値を再照合し、正しい金額で決済が行われるよう保護されています。
  • SQLインジェクション対策: データベースへのすべての操作は、WordPressの安全なクエリ発行手順（プリペイドステートメント）に従って行われています。
  • 不正操作の防止: 非ログインユーザーや権限のないユーザーが、受注データの編集や設定変更を行うことはできません。

28. よくある質問

Q. 購入者からメールが届いていないと言われた

管理画面の受注一覧から該当注文の「📧 再送」ボタンをクリックしてください。また、購入者に迷惑メールフォルダの確認もお願いしてください。

Q. ダウンロードリンクが切れていると言われた

購入者を注文照会ページ（[ec_order_inquiry]を設置したページ）に案内してください。注文番号 ＋ メールアドレスでダウンロードリンクを再取得できます。有効期限内であれば正常にダウンロードできます。また「マイページ」からも再取得可能です。

Q. Stripe決済でエラーが出た

設定画面でStripeシークレットキーが正しく入力されているか確認してください。テストモード（pk_test_...）と本番キー（pk_live_...）を間違えないよう注意してください。

Q. 送料が「住所入力後に表示」となっている

配送目的の商品がカートにある場合、配送先住所（都道府県）を入力することで送料が動的に計算されます。サブスクリプション商品で配送不要の設定にしている場合は、送料は発生しません。

Q. CSVをExcelで開いたら文字化けした

ファイルはUTF-8 BOM付きで出力されています。そのままExcelでダブルクリックして開いても文字化けしないようになっています。もし文字化けする場合は、Excelの「データ → テキストファイル」から読み込み、元のファイルの形式として「65001: Unicode (UTF-8)」を選択してください。

Q. 代金引換が選択できない

デジタル販売（ダウンロード商品）がカートに含まれている場合、またはお届け先地域によって代金引換をご利用いただけません。また、サブスクリプション商品が含まれる場合はクレジットカード決済のみとなります。

26. CSVでの商品一括管理

多数の商品データを一度に登録・更新するための機能です。Excel等で編集したCSVファイルをアップロードすることで、在庫数や価格、SKUなどを一括で変更できます。

📂 操作手順

  1. 販売商品 → CSV一括管理 メニューを開きます。
  2. 「CSVをダウンロード」をクリックして、現在のデータを取得します。
  3. Excel等でCSVを開き、内容を編集します。（新規登録したい場合は ID 列を空欄にします）
  4. 編集したファイルを「インポート」欄からアップロードして実行します。

📝 独自記法のルール

SKUやオプションなどの複雑なデータは、以下のルールで記述してください。

項目 記述ルール 例
バリエーション (SKU) 表示名=価格:加算額=在庫:数値=SKU:コード
※複数は | で区切る		 Sサイズ=価格:0=在庫:10=SKU:S001|Mサイズ=価格:500=在庫:5=SKU:M001
カスタムオプション グループ:名前=価格:加算額=形式:radio/check=必須:1/0
※複数は | で区切る		 ラッピング:希望=価格:500=形式:radio=必須:1|ラッピング:不要=価格:0=形式:radio=必須:1
カテゴリー 名前をカンマで区切る 風景, 2024年, 海
サムネイルURL 画像の直接URLを記載 自動的にダウンロード（サイドロード）され登録されます
💡 画像の最適化: すでにメディアライブラリにある画像URLを指定した場合、重複してダウンロードされないよう自動的に判定・スキップされる安全設計になっています。

PayPal連携の設定手順

PayPalを用いた決済を有効にするには、PayPal DeveloperダッシュボードからAPI資格情報を取得する必要があります。

🔑 APIキーの取得方法

  1. PayPal Developerにログインします。
  2. 「Apps & Credentials」から新しいREST API Appを作成します。
  3. 表示された Client ID と Secret をコピーします。
  4. 本プラグインの設定画面「決済設定」タブ内のPayPal項目に貼り付けます。
💡 Sandboxモードについて: テスト環境（Sandbox）で動作確認したい場合は、PayPal Developerで作成したSandbox用のキーを使用し、モードを「Sandbox」に設定してください。
⚠️ サブスクリプション非対応: 現バージョンのPayPal連携では「一括払い」のみ対応しています。カートにサブスクリプション商品（継続課金）が含まれている場合、PayPalは選択肢に表示されません。
🔄 キャンセル時の挙動: PayPalの決済画面で「キャンセルしてショップに戻る」を選択した場合、注文は確定されず、カートの内容を保持したままチェックアウト画面に戻ります。
✨ 高信頼なサンクスページ [v5.1.0]: 決済完了後の画面表示と確定処理を分離する最新の設計（冪等性確保）を採用しています。これにより、万が一ブラウザをリロードしたり、通信が不安定な場合でも、二重決済を防ぎつつ確実に注文詳細が表示され、メールが届くようになっています。
