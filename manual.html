1. プラグイン概要

このプラグインはWordPress上で汎用的な商品を販売するための機能をすべて提供します。

✅ 主な機能

機能 説明
商品販売 ダウンロード版・配送品の2形式に対応
決済 クレジットカード・PayPay（Stripe）・銀行振込・代金引換
クーポン 割引コード（定額・定率）の発行と管理
注文管理 受注一覧から全注文を一括管理
メール通知 注文確認・キャンセル・発送通知メールを自動送信
ダウンロード管理 デジタル販売の有効期限・回数制限
CSV出力 売上データをCSVエクスポート（確定申告対応）
注文照会 購入者が自分で注文状況を確認できるページ
送料計算 住所入力後に自動計算（通常配送商品のみ）
在庫アラート 設定した在庫数を下回った際に管理者にメール通知

2. 初期設定

WordPress管理画面 → 各種商品 → 各種設定 から設定を行います。

📌 設定項目（各種設定タブ）

タブ 項目 説明
一般設定 表示用店舗名 注文メールや領収書の送り主名として使用されます（初期値：サイト名）
決済設定 決済方法のON/OFF カード、PayPay、銀行振込、代金引換の利用可否を切り替えます
Stripe APIキー / Webhook秘密鍵 カード・PayPay決済に必要な公開可能キーとシークレットキーを設定します。
サブスクリプションの解約同期や安全な通信（署名検証）にはWebhook秘密鍵の設定が必須です。
デジタル販売設定 ダウンロード商品の有効期限（日数）と上限回数を設定できます。
0を指定すると無期限・無制限になります。
代引き手数料 代金引換が選択された際に加算される手数料を設定します。
配送総額に応じた段階料金（ティア設定）を自由にカスタマイズ可能です。
支払い/特商法 標準税率 / 軽減税率 標準税率（10%）と軽減税率（8%）の数値を設定します
銀行口座 お振込先案内 銀行振込が選択された際に購入者へ案内する口座情報を登録します
送料設定 全国一律送料 基本となる送料（円）を設定します
特定商取引法 各表示項目 ネット販売に必須の特定商取引法に基づく表記を編集します（販売形式がサブスクリプションの場合は、課金周期や解約方法の追記が義務付けられています）。[ec_tokushoho]で表示可能です
登録番号 (T番号) 適格請求書発行事業者登録番号（T番号）を設定します。
設定すると、各通知メール、受注詳細画面、印刷用納品書・領収書に自動的に表示され、適格請求書（インボイス）としての要件を満たします。

3. 商品の登録・管理

各種商品 → 商品を新規追加
タイトル、商品ファイル、サムネイル画像、販売形式と価格を設定します。
在庫管理の設定
「在庫管理を行う」にチェックを入れると、在庫数に基づいて販売を制限。0になると自動で「売り切れ」表示になります。

4. メディアライブラリからの一括登録

数十点、数百点の商品を効率よく登録するための機能です。

📝 手順

  1. WordPressメニュー：各種商品 → 一括登録 を開く
  2. 「メディアライブラリから選択」をクリックし、画像を選択
  3. カテゴリーと各形式の初期価格を設定して「商品を一括生成する」をクリック

6. 注文管理（受注一覧）

WordPress管理画面 → 各種商品 → 受注一覧 で全注文を管理します。

📋 注文のフィルタリング

上部のタブで表示する注文を切り替えられます。

  • 通常注文: 未決済の電子決済（Abandoned）を除いた、有効な注文のみを表示します。
  • 放置注文: 決済が完了していないStripe/PayPay注文、または7日以上経過した入金待ち注文を抽出します。
  • すべて表示: システム上のすべての注文を表示します。

⚙️ ステータスの一括更新

複数の注文を選択して、一括でステータスを変更できます。

  1. 受注一覧で対象の注文にチェックを入れる
  2. 上部または下部の「一括操作」から以下を選択：
    • ステータス: 準備中に変更
    • ステータス: 完了（発送済み）に変更
  3. 「適用」ボタンをクリック

※ステータス変更時には、購入者へ自動的に「ステータス更新通知メール」が送信されます。

🗑️ 選択して一括削除

「一括操作」から「削除」を選んで「適用」をクリックすることで、複数の注文をまとめて削除できます。この操作は元に戻せませんのでご注意ください。

7. 注文ステータスフロー

ステータス 意味 次のアクション
入金待ち 銀行振込の入金確認待ち 入金確認後 → 「決済完了」へ変更
決済完了 / 発送待ち 決済が完了し、配送準備中 発送完了後 → 「発送済み」へ変更
🟣 サービス中 / 継続中 サブスクリプション（定期購入）が有効な状態 次回の自動課金を待ちます。解約されるまでこの状態が続きます。
発送済み / 完了 商品を発送済み、または単発サービスの提供完了 —（完了）
キャンセル 注文をキャンセルした 購入者に自動でキャンセルメールが送られます
💡 クレジットカードまたはPayPay（Stripe）決済の場合、決済が完了した時点で自動的に「決済完了 / 発送待ち（サービス・デジタルの場合は『サービス中 / 継続中』）」になります。

6. 各種アクション操作

編集 — 注文を編集

購入者情報・ステータス・送り状番号を変更できます。送り状番号を入力して保存すると、購入者への発送通知メールが自動送信されます。

📧 再送 — 確認メール再送

注文確認メールを購入者に再送します。メールが届いていないと連絡を受けた際に使用します。

キャンセル — 注文キャンセル

注文をキャンセルします。実行すると：

  • ステータスが「キャンセル」に更新される
  • 購入者にキャンセル通知メールが自動送信される
  • クレジットカード・PayPay（Stripe）決済の場合、自動的に全額返金が実行されます。
💡 自動返金の結果は画面上のメッセージで確認できます。銀行振込や代金引換の場合は、手動での返金対応が必要です。
💡 在庫とクーポンの復元：キャンセルを実行すると、その注文に含まれていた商品の在庫および使用されたクーポンの利用回数が自動的に復元（差し戻し）されます。
⚠️ キャンセル後に「発送済み」などに戻すには「編集」からステータスを変更してください。

📄 納品書 / 🧾 領収書 — 各種帳票出力

注文詳細に基づいた納品書や領収書を「印刷用A4レイアウト」で表示します。ブラウザの印刷機能を使用してPDF保存や紙への印刷が可能です。

  • 納品書：商品明細や配送先を記載します。
  • 領収書：合計金額を強調した領収書形式で出力します。

削除 — 注文削除

注文を完全に削除します。この操作は元に戻せません。通常はキャンセルを使用し、削除は不要な注文のみに限定することを推奨します。

💡 在庫・クーポンの復元：未キャンセルの注文を削除した場合、削除前に在庫およびクーポン使用回数が自動的に復元されます。

8. クーポン設定・管理

WordPress管理画面 → 各種商品 → クーポン管理 でクーポンを作成・管理できます。

🎫 クーポンの作成

「新規追加」から以下の項目を設定してクーポンを発行します。

項目 説明
クーポンコード お客さまが入力する半角英数字のコード（例: WELCOME2024）
割引種別 百分率(%) または 固定額(円) を選択
割引額 10%なら「10」、500円なら「500」と入力
有効期限 クーポンの利用期限を設定（未設定で無期限）
利用回数上限 そのクーポンが全体で何回使われたら無効にするかを設定（未設定で無制限）

🛒 お客さま側の利用方法

チェックアウト（お支払い）画面の「クーポンをお持ちですか？」をクリックし、コードを入力して「適用」ボタンを押すと、リアルタイムで合計金額が更新されます。

💡 利用回数の自動カウント：注文が確定するたびに、利用されたクーポンの「現在の利用回数」が自動で加算されます。上限に達したクーポンは自動的に無効となります（※注文キャンセル時には自動的に回数が差し戻されます）。

9. サブスクリプション機能 [v3.0.0]

継続的なサービス（月額・年額など）を販売するための機能です。

📦 商品をサブスクとして設定する

「商品を新規追加」または「編集」画面で、「この商品をサブスクリプションとして販売する」にチェックを入れます。

💡 サブスクを有効にした商品は、Stripe Checkout で自動的に「継続課金」として処理されます。
🚚 配送が必要な商品 (定期購入)にチェックを入れると、チェックアウト時に住所入力が必須になります（サービス提供型の場合はチェックを外してください）。
⚠️ サブスクリプション商品が含まれる場合、クレジットカード決済のみとなります（PayPay・銀行振込・代金引換はご利用いただけません）。

💳 購入者の管理ポータル

購入者は以下の方法で自身のサブスクリプションを管理（解約、カード変更など）できます。

  • 注文照会ページ: 照会結果の画面に表示される「サブスクリプション管理」ボタンから移動。
  • 専用ショートコード: [ec_customer_portal] を設置したページから（登録メールアドレスへの認証コード送信による安全なログインが可能です）。
⚠️ Webhookの設定：サブスクの「解約」をシステムに反映させるには、Stripe管理画面でWebhookの設定（customer.subscription.deleted を「送る」設定）が必要です。

10. 購入者向け注文照会ページ

購入者が自分で注文状況を確認できるページです。ショートコードを固定ページに設置するだけで動作します。

設置方法

固定ページを新規作成 例：「注文照会」というタイトルで固定ページを作成
ショートコードを貼り付け
[ec_order_inquiry]
確認メールにURLを追記 設定の確認メール文面に「注文照会はこちら：（ページURL）」を追加

照会ページでできること

  • 注文番号 ＋ メールアドレスで注文詳細を確認
  • 注文ステータス（入金待ち / 発送準備中 / 発送済み / キャンセル）の確認
  • 送り状番号の確認（発送後に表示）
  • デジタル商品のダウンロードリンク再取得
  • インボイス対応の税額内訳表示（消費税 10% / 8% の対象額と税額を表示）
💡 注文番号は確認メールに記載されています。購入者に案内してください。

💬 注文備考（メッセージ機能）

購入者がチェックアウト時に管理者へメッセージを残すことができます。

  • 購入者側: 支払い画面の「備考」欄に自由に入力できます。
  • 管理者側: 各通知メールおよび「受注編集」画面で内容を確認できます。
  • 追記・修正: 管理者は「受注編集」画面から備考内容を更新することも可能です。

8. ダウンロード管理

ダウンロード版の購入の場合、リンクに有効期限と回数制限を設定できます。

設定項目 デフォルト 説明
ダウンロード有効期限 7日間 注文日から何日間ダウンロード可能か
ダウンロード上限回数 5回 一つの注文で何回ダウンロードできるか
⚠️ 有効期限・上限回数を超えた場合、購入者が注文照会ページから確認すると「期限切れ」または「上限に達しました」と表示されます。延長が必要な場合は管理画面から注文を編集してください。

9. 受注データのCSVエクスポート

受注一覧の右上にある「💾 CSVエクスポート」ボタンをクリックすると、全注文データをCSVファイルでダウンロードできます。

CSVの項目

注文ID / 注文番号 / 購入者名 / メールアドレス / 購入商品 / 商品小計 / 送料 / 代引き手数料 / クーポンコード / クーポン割引額 / 10%対象額 / 10%消費税 / 8%対象額 / 8%消費税 / 合計金額 / 支払い方法 / ステータス / 送り状番号 / 注文日時 / 郵便番号 / 都道府県 / 住所 / Stripe 顧客ID / Stripe サブスクID

使用シーン

  • 確定申告時の売上集計
  • Excelでの売上分析
  • バックグラウンドでの顧客管理
✅ ファイルはUTF-8 BOM付きで出力されるため、Excelで開いても文字化けしません。

10. 商品CSV一括編集 [v3.7.0]

多数の商品の価格や在庫状況、サブスクリプション設定をCSVファイルを使って一括で更新できる機能です。

📝 操作手順

  1. 各種商品 → 商品CSV一括編集 に移動します。
  2. 「現在のデータをダウンロード」ボタンを押し、CSVファイルを保存します。
  3. Excel等でCSVを開き、価格や在庫数を編集して保存します。
  4. 「CSVファイルを選択」から編集したファイルをアップロードし、「インポートを実行」をクリックします。

📁 カラム（列）の設定ルール

項目ルール・説明
IDシステム識別用（変更不可）
商品名タイトルの変更が可能です。
各種価格半角数値で入力してください。
税率区分standard(10%) / reduced(8%)
各種フラグ1で有効、0で無効（売り切れ、在庫管理、サブスク等）
サブスク、サイクルday(日) / week(週) / month(月) / year(年)
💡 Excel対応：Shift-JISおよびUTF-8の両方のエンコーディングに対応しているため、日本のExcelでそのまま編集して保存しても正しくインポートされます。
⚠️ 画像について：現在のバージョンでは画像の追加・変更には対応しておりません。画像はWordPressの管理画面から個別に設定してください。

11. 在庫アラート機能 [v3.7.1]

在庫切れによる販売機会の損失を防ぐため、在庫が少なくなった際に自動で管理者に通知する機能です。

🚨 通知の仕組み

  • しきい値の設定: 「設定 > 一般」タブの「在庫アラートのしきい値」で、何個以下になったら通知するかを設定できます（デフォルト: 5）。
  • リアルタイム判定: 購入によって在庫が減った際や、CSV・手動で在庫を書き換えた際に自動チェックが行われます。
  • メール通知: 在庫がしきい値以下になると、管理者の受信メールアドレスに「【在庫不足アラート】商品名」という件名のメールが届きます。
  • 重複防止: 一度数値が下がって通知された後は、在庫が補充（しきい値より大きく設定）されるまで、同じ商品のアラートは二重送信されません。
💡 運用のコツ
人気商品はしきい値を多め（例: 20個）に設定し、一点ものや入荷が早いものは少なめ（例: 2個）に設定したしきい値を「一般設定」で調整してください。

13. クーポン管理

割引クーポンを作成・管理できます。各種商品 → クーポン管理 で操作します。

🎫 クーポンの作成

新規追加または編集画面で以下の項目を設定できます。

  • クーポンコード: お客様がカート画面で入力するコードです。
  • 割引額/率: 定額（円）またはパーセント（%）を設定します。
  • 最低購入金額: この金額以上の注文にのみクーポンが適用されます。
  • 利用期限 / 回数制限: クーポンの有効期限や、全体での使用回数上限を設定できます。

🔄 サブスクリプションと割引期間

サブスクリプション商品（定期購入）に対してクーポンがどのように適用されるかを設定できます。これはStripe決済時に自動的に反映されます。

  • 初回のみ (1回限り): 初回の支払いのみ割引が適用され、2回目以降は通常価格になります。
  • 永続: 解約されるまで、毎回の支払いにずっと割引が適用されます。
  • 指定した期間: 「3ヶ月間のみ割引」のように、指定した回数分だけ割引を継続させることができます。

※ カート画面では、サブスクリプション商品にクーポンが適用された際、その期間（初回のみ、等）が明示されます。

14. よくある質問

Q. 購入者からメールが届いていないと言われた

管理画面の受注一覧から該当注文の「📧 再送」ボタンをクリックしてください。また、購入者にスパムフォルダの確認もお願いしてください。

Q. ダウンロードリンクが切れていると言われた

購入者を注文照会ページ（/order-inquiry/）に案内してください。注文番号 ＋ メールアドレスでダウンロードリンクを再取得できます。有効期限内であれば正常にダウンロードできます。

Q. Stripe決済でエラーが出た

設定画面でStripeシークレットキーが正しく入力されているか確認してください。テストモードのキーと本番キーを間違えないよう注意してください。

Q. 送料が「住所入力後に表示」となっている

配送が必要な商品がカートにある場合、配送先住所を入力することで送料が自動計算されます。サブスクリプション商品で配送不要（サービス提供のみ）の設定にしている場合、送料は発生しません。

Q. CSVをExcelで開いたら文字化けした

ファイルはUTF-8 BOM付きで出力されています。Excelで「データ → テキストファイル」から読み込む場合はエンコード「UTF-8」を選択してください。

Q. 代金引換が選択できない

ダウンロード版商品（デジタル販売）がカートに含まれている場合、またはお届けエリア（離島など）によっては代金引換をご利用いただけません。その場合はクレジットカード、PayPay、または銀行振込をご利用ください。

9. セキュリティと信頼性

本プラグインは商用利用を前提として、以下のセキュリティ対策を講じています。

  • 価格操作の防止: チェックアウト時にブラウザ側で商品価格やオプション価格が書き換えられた場合でも、サーバー側でデータベースの値を再照合し、正しい金額で決済が行われるよう保護されています。
  • SQLインジェクション対策: データベースへのすべての操作は、WordPressの安全なクエリ発行手順（プリペアドステートメント）に従って行われています。
  • 不正操作の防止: 非ログインユーザーや権限のないユーザーが、受注データの編集や設定変更を行うことはできません。
